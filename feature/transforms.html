Transforms

General-purpose feature transformations, one file per transform under qrt/feature/transforms/. These code cells are executed live by Quarto every time the docs are built.

Sample data

We use qrt’s bundled AAPL sample dataset — loaded offline via q.data.datasets.load, no network dependency (see the Data tutorial for more on q.data):

import pandas as pd

import qrt as q

aapl = q.data.datasets.load("aapl")

Build model-ready columns with the transforms provided by q.feature.transforms:

sma_20 = q.feature.transforms.sma(aapl["close"], 20)
sma_100 = q.feature.transforms.sma(aapl["close"], 100)

price_features = pd.DataFrame({"close": aapl["close"], "sma_20": sma_20, "sma_100": sma_100})

To add a transform, create qrt/feature/transforms/_<transform>.py with one public function and re-export it in qrt/feature/transforms/__init__.py (one import line + one __all__ entry).

Visualizing features

q.plot.col renders any feature DataFrame as an interactive Plotly chart with hover, zoom, and range-selector buttons — handy for eyeballing a feature against price before using it in a model:

fig = q.plot.col(
    price_features,
    title="AAPL close price with SMA overlays",
    ylabel="Price",
    height=450,
)
fig.show()
