Feature Engineering

This page’s code cells are executed live by Quarto (via Jupyter) every time the docs are built. q.feat collects feature engineering tools under one namespace, organized as submodules:

Submodule What it is
q.feat.qta our own hand-rolled features, one file per feature
q.feat.talib every TA-Lib indicator, pandas-friendly
q.feat.pandas_ta every pandas-ta-classic indicator

All indicator functions share the same convention: they accept an OHLCV DataFrame with lowercase columns (open, high, low, close, volume) — the frame layout returned by q.data.sources’s read() functions (e.g. q.data.sources.yfinance.read(...)) — and preserve the datetime index on the output. Indicators that only need a price series also accept a plain Series (treated as close).

qta — our own features

Hand-rolled primitives, one file per feature under qrt/feat/qta/:

sma_20 = q.feat.qta.sma(aapl["close"], 20)
sma_100 = q.feat.qta.sma(aapl["close"], 100)

price_features = pd.DataFrame({"close": aapl["close"], "sma_20": sma_20, "sma_100": sma_100})

To add a feature, create qrt/feat/qta/<feature>.py with one public function and re-export it in qrt/feat/qta/__init__.py (one import line + one __all__ entry).

Visualizing features

q.plot.col renders any feature DataFrame as an interactive Plotly chart with hover, zoom, and range-selector buttons — handy for eyeballing an indicator against price before using it in a model:

fig = q.plot.col(
    price_features,
    title="AAPL close price with SMA overlays",
    ylabel="Price",
    height=450,
)
fig.show()

TA-Lib

Single-output indicators return a Series named after the indicator; multi-output indicators return a DataFrame with TA-Lib’s output names as columns:

rsi = q.feat.talib.RSI(aapl)
atr = q.feat.talib.ATR(aapl, timeperiod=20)
macd = q.feat.talib.MACD(aapl)

pd.concat([rsi, atr, macd], axis=1).tail()

IDE support

The talib and pandas_ta wrappers create their functions dynamically at runtime. Autocomplete and hover docs are provided by generated type stubs — regenerate them after upgrading either library:

make stubs
